01
哈希两把刀 这题不设计 key——set 是哈希的另一半本事
工具干什么见过的题
dict (key→value)按签名分组 / 计数 / 查表LC1 两数之和、LC49 异位词
set (只有 key)O(1) 判断"在不在"本题 LC128
  • set = 只留钥匙、不挂东西的字典。只回答一个问题:x in s?——瞬间答你。对比 x in list 要从头扫到尾(O(n))。
  • 别把 LC49 的"哈希=设计key"当万能定势——"在不在"类问题,set 一把梭,不需要任何 key 设计。
  • 零件:set(nums) 一行建集(与 list() 同族);塞入用 .add() 不是 append;重复元素自动只留一份。
口诀分组计数找 dict,判断存在找 set——哈希家族两把刀,认清题型再拔刀。
02
核心洞察 只从"一串的起点"开始数,O(n²) 变 O(n)
  • 题目卡 O(n),就是冲着"不许排序"来的(排序 O(n log n),人人都会,不考你这个)。
  • 天真版:每个数都往右数 num+1、num+2…——[1,2,3,4,5] 里 1 走到 5、2 又走到 5、3 又走……每个数重走后缀,O(n²) 假货。
  • 题眼:n-1 不在 set 里 ⇒ n 左边没邻居 ⇒ n 是一串的头。只有头才有资格起跑;非头一票否决直接跳过。每个数最多被走到一次 → 真 O(n)。
nums_set = set(nums)
max_length = 0
for n in nums_set:              # 遍历 set,重复数字不白查
    if n - 1 not in nums_set:   # 起点判据只有这一条
        temp, length = n, 1
        while temp + 1 in nums_set:
            temp += 1            # 人往前走
            length += 1
        max_length = max(length, max_length)
return max_length
套路线性扫描怕重复劳动,就只从每段的起点干活——判起点的代价 O(1),省下的是整段重走。
03
血泪坑 · TLE while 的条件变量,循环体内必须有人推它
  • 翻车现场:while temp + 1 in nums_set: 循环体里只写了 length += 1,temp 一步没动 → 条件永远为真 → 死循环 → LeetCode 判 TLE(超时)。
  • 脑子里想的是"沿着 n+1、n+2 往右走",但代码里"走"的那句(temp += 1)没写——意图和代码差一句,就是死循环。
  • 通用体检项:写完任何 while,盯着条件问一句——条件里的变量,循环体内谁在推它? LC7 的推进器是 var //= 10,本题是 temp += 1。
口诀while 必有推进器。没人推条件变量,循环就转到天荒地老。
04
补丁链复盘 一个病根引出两块膏药,拔根它们全下岗 
病根: if 里多写了 and n+1 in nums_set  → 孤点[5]被挡在外
  ↓ 症状: [5] 返回 0
膏药1: return max(max_length, 1)      → [5] 对了,但空数组变 1
  ↓ 新伤: [] 返回 1
膏药2: if len(nums)==0: return 0      → 救膏药1
三块互相搀扶,拆任何一块全塌。拔掉病根 → 两块膏药同时下岗。
  • 起点的定义只有一条:n-1 不在 set。右边有没有人是 while 该发现的事,不是入场券——多加的半句把长度为 1 的序列判了死刑。
  • "改到绿为止"是要戒的瘾:每次 WA 都在指认病根,你却在出口调膏药配方。看懂为什么红,才是本事。
套路修 bug 拔根因,别用补丁供养补丁——补丁需要新补丁来救,就是走错路的铁证。
05
空数组之辩 不用单独 if——for 的零圈天然吞掉它
  • 把 [] 走一遍:set([]) 不炸 → for 遍历空 set 零圈,静默跳过 → 返回初始值 0,恰是正确答案。没有任何一步需要救。
  • Python 里空容器遍历不报错、不溢出,危险只有一种:你以为它走了某段逻辑而它没走 → 后果是 WA(答案错),不是 crash(程序崩)。"溢出"只属于定宽整数算术,别当万能怕点。
  • 什么时候才真要判空?判据:空输入走正常逻辑会不会炸或算错。nums[0]、sum/len 会炸——挡;for 遍历、本题——天然正确,挡它就是画蛇添足。
口诀好算法把边界当普通情况顺路吞掉,而不是门口摆一排 if。防御要防真枪,不防空气。
