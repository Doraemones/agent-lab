01
真考点 反转是白给的，面试官埋的雷是溢出
  • 反转本身三行搞定（%10 剥末位、//10 削末位、拼回去）。真考点是 32 位有符号整数溢出。
  • 范围 [−2³¹, 2³¹−1] = [−2147483648, 2147483647]。反转会撑爆，比如 1534236469 反转后越界，要求返回 0。
  • 隐藏清单必须自己报出来：负数、末尾零（120→21）、溢出边界、INT_MIN 本身。
口诀题面写"反转"，考的是边界思维：不满足于正常输入能跑，逼问极端情况会怎样。
02
数值法核心 剥一位、拼一位，末尾零自动消失
  • 核心就一行递推：result = result * 10 + digit。每来一个新数字，老的 result 乘 10"腾位"，再把新数字塞进个位。
  • trace 123：0→3→32→321。%10 取末位、//10 削末位，削到 0 就结束。
  • 末尾零是"字符串法自己造出来的假问题"：数值法里 120 剥出 0 → result 还是 0，自然没了，什么都不用管。
result = 0
while var:
    digit = var % 10          # 剥末位
    result = result * 10 + digit
    var //= 10               # 削末位（注意是 //）
口诀选一个让边界情况自己消失的表示法，别选一个制造边界再打补丁的。数值法让末尾零凭空消失。
03
血泪坑 / 与 // 之别，以及 Python 2 已经死了
Python 3Python 2（已 EOL）
5 / 22.5 永远浮点2 偷偷整除！
5 // 22 整除2 整除
  • 本题栽的就是 var /= 10：Py3 里它把 var 变成浮点，digit 变小数、var 永远除不到 0、result 滚成 inf，全废。要整除就明写 //=。
  • 那个曾用 /2.0 强凑浮点的旧招，是在骗 Py2 的 /；到 Py3 直接用 / 就是浮点，扔掉。
  • Python 2 已于 2020 年 EOL——新代码一律 Py3，LeetCode 语言框选 Python3 不是 Python。
口诀别写"靠解释器版本才对"的代码。要整除写 //，要浮点写 /，意图一眼可读，哪个 Python 上都对。
04
溢出的两副面孔 同一道题，两种语言两套解
Python
整数任意精度，永不溢出、永不报错、永不失真。→ 事后检查：算出真值 result，再比 [−2³¹, 2³¹−1]，越界返 0。
C++ / Java
int 是定宽 32 位盒子，越界当场坏成垃圾值。→ 事前拦截：必须在 result*10+digit 之前判断会不会越界。
  • Python 那句越界比较，目的不是"防程序崩"，而是模拟一个 32 位世界的业务规则——它不会崩，只是题目要求越界吐 0。
口诀"溢出"在不同语言是两种东西：Python 事后比，C++ 事前拦。懂了整数模型，才算真懂溢出。
05
事前拦截 C++ 移植版：乘之前就把溢出摁住
  • 把边界拆开看：2147483647 = 214748364 * 10 + 7。乘 10 加 digit 之前，先问这一步会不会破。
  • 两段式条件：result > limit//10，或 result == limit//10 且 digit > limit%10，命中就 return 0。
  • 魔鬼细节——正负边界差 1：正数上限末位是 7（…647），负数取绝对值末位是 8（…648）。limit%10 自动兜住。
if result > limit // 10 \
   or (result == limit // 10 and digit > limit % 10):
    return 0
result = result * 10 + digit   # 拦过才敢乘
口诀纯 Python 用不上它，但面试要求"别靠大整数"、或要移植 C++/Java 时，它就是正解。两版都会，才知道何时用哪版。
06
洞悉核心 + 好习惯 比这道题更值钱的三条
  • 先 abs 拆符号：Python 负数的 %/// 反直觉（-123%10=7、// 向负无穷取整）。用 abs 转正数处理、sign 最后贴回，一次绕开所有负数坑。
  • 拿到 review，先修 🔴 致命，再碰 🟡 小尾巴。这次改名换行都做了、却漏了唯一让代码跑不起来的 //=——反了。
  • 别过早优化：想用"字符串长度判溢出"是自作聪明，一句 O(1) 的越界比较就够，分支只添 bug。
心法高手的简洁不是写得花，是让麻烦没机会出现：选对表示法、明写意图、先解决最要命的那一个。
