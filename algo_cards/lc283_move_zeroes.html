01
视角转弯 不是"移动 0"，是"把非零码齐，剩下自然是 0"
  • 要求:所有 0 挪到末尾、非零相对顺序不变、且原地(不新建数组)。
  • 暴力直觉是"看到 0 就 del、末尾补 0"——但边遍历边删会下标错位,得靠 i-=1 打补丁,还得计数,而且 del 是 O(n) → 整体 O(n²)。这是要撞的墙。
  • 换视角:你不搬 0，你把非零往前码；码完，后面的位置天然就是 0 该在的地方。难题即化——和 LC128"从起点码序列"同一种转弯。
口诀原地重排别想着"删"，想着"码"：把要留的往前写，剩余位置统一收尾。
02
读写双指针 同一个数组，两个下标，O(1) 空间
  • read 往前扫找非零;write 标记"下一个非零该落的位置"。两个都是 nums 的下标，不是新数组。
  • read 每遇一个非零 → 写到 nums[write] → write 前进一格。扫完,write 到末尾全填 0。
  • 为什么原地覆盖不丢数据?铁律 write ≤ read 恒成立:你盖的位置永远是已读过的(要么早copy过的非零,要么乐意踩掉的0),绝不盖到没读的元素。
nums=[0,1,0,3,12]write=0
read→1 (非零)nums[0]=1 → [1,1,0,3,12], write=1
read→3 (非零)nums[1]=3 → [1,3,0,3,12], write=2
read→12 (非零)nums[2]=12 → [1,3,12,3,12], write=3
填 0 (write→末尾)[1,3,12,0,0] ✓
write = 0
for read in range(len(nums)):   # read 直接当循环变量,别再单独 i
    if nums[read] != 0:
        nums[write] = nums[read]
        write += 1
for k in range(write, len(nums)):
    nums[k] = 0
套路"原地重排/搬移"类题 → 读写(快慢)双指针:read 扫、write 收，同一数组两下标，O(1) 空间。
03
验收扣分 AC 不等于干净：两处"没读合同"
  • 重复计数器:同时写 for i in range(...) 又手动 read += 1,read 永远等于 i，雇了两遍同一个人。留一个:直接 for read in range(...)。
  • return 违约:签名写 -> None、题目大字"Do not return anything"，代码却 return nums。LeetCode 不看返回值才蒙混 AC，但 mypy 会报错、reviewer 会问。
  • 教训:函数签名 + 文档 = 合同，合同说啥照做。这跟 LC7 漏看"32位约束"是同一种"没读清要求"的病。
心法AC 只证明"能跑"，不证明"干净"。签名和 docstring 是白纸黑字的合同，别言行不一。
