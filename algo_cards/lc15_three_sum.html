01
核心套路 kSum 降维：排序 → 固定 k-2 个 → 剩 2 个对撞
  • kSum 通式:排序后固定 k-2 个数,把剩下 2 个交给对撞双指针。2Sum 固定 0、3Sum 固定 1、4Sum 固定 2……一个模板吃一整类。
  • 3Sum:固定 nums[left],右边就退化成"有序数组找两数之和 = -nums[left]",正是 LC11 那种左右对撞。
  • 为什么必须排序?因为你按"和"找配对——移动指针要让和单调变化,这只有有序才成立(对比 LC11 按"宽度"找,天生有序、不用排)。
套路kSum 降维:排序 + 固定 k-2 个 + 对撞双指针。这条通式是这一整类题的万能钥匙。
02
主体骨架 和大了收右，和小了收左，命中就记一笔 
nums.sort()
res = []
for left in range(len(nums) - 2):
    if left > 0 and nums[left] == nums[left - 1]:   # 去重①:固定数
        continue
    mid, right = left + 1, len(nums) - 1
    while mid < right:
        s = nums[mid] + nums[right]
        if s == -nums[left]:
            res.append([nums[left], nums[mid], nums[right]])
            mid += 1; right -= 1
            while mid < right and nums[mid] == nums[mid - 1]:   # 去重②
                mid += 1
            while mid < right and nums[right] == nums[right + 1]: # 去重③
                right -= 1
        elif s > -nums[left]:
            right -= 1       # 和太大,收右
        else:
            mid += 1        # 和太小,收左
口诀时间 O(n²)、空间 O(1)。排序的 O(n log n) 被 n² 吃掉,等于免费——所以这题排序稳赚。
03
血泪坑 · 守卫 守卫判"下标范围"，绝不判"值真假"
  • 3Sum 比 2Sum 难的唯一原因是去重——结果不能有重复三元组,三处都要跳:固定数、mid、right。
  • 翻车写法:if nums[left-1] and ... 拿"值真假"当守卫。一次炸三雷:①没边界判断 → IndexError;②0 是假值 → 该去重的静默关掉([0,0,0,0] 出重复);③left=0 时 nums[-1] 绕回末尾。
  • 正解:守卫一律判下标范围——left > 0、mid < right,不是 nums[x]。
心法去重/边界的守卫,判的永远是"下标在不在范围内",不是"值真不真"。这是本题最贵的一课。
04
去重两套写法 各自自洽，绝不能一套取一半

A 套 · 先前进再回头看

mid += 1
while ... and \
  nums[mid]==nums[mid-1]:
    mid += 1
# 比"身后刚用过的值"

B 套 · 先向前看再前进

while ... and \
  nums[mid+1]==nums[mid]:
    mid += 1
mid += 1
# 比"身前下一个值"
  • 两套都对,但混着用必崩:曾把 A 的"先前进"配 B 的"看 nums[mid+1]",前进后却往身前看 → 查错参照物 → 没跳掉刚用过的值 → [-2,0,0,2,2] 吐重复。
  • 病根:前进之后,要跳的是"跟刚用过的值相同"的元素,而它现在在你身后(nums[mid-1]),不是身前。
口诀去重模板是一个整体:要么 A 套要么 B 套,别拆开重拼——参照物错一步,去重全废。
05
控制流真相 守卫是"检查点"，不是"持续监控"
  • 外层 while mid < right 只在每轮开头查一次,进了循环体就撒手。内层去重疯狂挪指针时,它全程不在场。
  • 所以内层去重必须自带 mid < right。反例 [0,0,0,0,0]:不加内层守卫,right 从 4 一路减穿 mid、减到 0、再 nums[-1] 绕回 → 你担心的 "right-1=-1" 真的发生。
  • 为什么加了就够?靠不变量 mid ≥ 1(因 mid=left+1):mid<right 且 mid≥1 ⇒ right≥2 ⇒ right-1≥1,永远越不到 -1。
心法循环条件是检查点式的,不是持续监控。哪个循环改指针,守卫就装在哪个循环头上;边界安全要靠追不变量证明,别盲信。
06
复盘 + 好习惯 一道 Hard 从 IndexError 调到干净 AC
  • 别拿内置名当变量:list = [](第 2 次,LC49 已点)会盖掉内置 list()。改 res。
  • 三个边界追问值千金:你连问"外层判了内层为啥还判""right-1 会不会 -1""mid<right 够不够"——从"会写"到"吃透"就差这种较真。
  • 抽象到通式:你把套路总结成"固定 k-2 个数",不是"固定 1 个"——一句话吃下整个 kSum 家族,这才是套路的正确沉淀方式。
套路沉淀kSum 降维 = 排序 + 固定 k-2 个 + 对撞双指针;去重三处,守下标不守值真假。
