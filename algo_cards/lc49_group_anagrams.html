01
题意翻译 异位词 = 同一把字母，打乱重排
  • 字母异位词（anagram）：两个词用的字母完全一样、只是顺序不同。eat / tea / ate 都是 a·e·t，互为异位词。
  • 任务：把互为异位词的词分进同一组。["eat","tea","tan","ate","nat","bat"] → [["eat","tea","ate"],["tan","nat"],["bat"]]。
  • 组内顺序、组间顺序都不作要求——就是按"用了哪些字母"给单词分堆。
口诀题面吓人的名词先翻译成人话：异位词分组 = 按字母构成分堆。
02
核心套路 分组问题 = 造签名 + 哈希表装筐
  • 难点只有一个：给异位词造一个"签名（key）"，让 eat/tea/ate 算出来一模一样。
  • 答案：把字母排序。sorted("eat") → "aet"，sorted("tea") → "aet"——排序把"顺序"这个差异抹掉，只剩"构成"。
  • 然后一层循环：算签名 → 进对应的筐 → 最后交出所有筐。sorted() 返回列表，须 "".join(...) 拼回字符串——列表不能当 dict 的 key，字符串可以。
groups = {}
for word in strs:
    key = "".join(sorted(word))   # 签名："eat"→"aet"
    if key not in groups:
        groups[key] = []             # 新签名 → 建空筐
    groups[key].append(word)         # 统一塞筐，不写 else
return list(groups.values())
套路遇到分组类问题：给每个东西设计一个"归一化签名"，签名相同进同一筐，筐用 dict[key] → list 装。
03
dict 铁律 一个 key 一个 value；想一对多，value 当筐
  • 字典铁律：key 不重复，一个 key 永远只挂一个 value。不能往 "aet" 下塞三个独立的词。
  • 诀窍：value 不是词，是一个列表（筐）。塞进字典的始终只有一个 value——一只筐；变多的是筐里的内容。
  • "筐里的词怎么区分？"——不需要区分。互为异位词本来就该混在一起，这正是分组的目的。dict 只负责"不同签名进不同筐"这一层。
groups = {
    "aet": ["eat", "tea", "ate"],   # 1 个 key → 1 个 value（装3个词的筐）
    "ant": ["tan", "nat"],
    "abt": ["bat"],
}
口诀dict[key] → list 这个形状以后无处不在：按类别分组、按用户攒消息、图的邻接表——今天焊牢。
04
血泪坑 for x in 字典 拿到的是 key，不是 value
写法遍历出什么
for k in groups:key（签名）——本题的翻车点，交上去一堆 "aet"
for v in groups.values():value（筐）——本题要的
for k, v in groups.items():key 和 value 一起
  • 同族坑：for x in 列表 得元素（整个词），for x in 字符串 得单个字符。遍历出什么，取决于容器是什么——给 word 起名 ch，就是这个误会。
  • 命名要如实：装着 "eat" 的变量叫 ch，三个月后的自己必懵。
口诀字典默认吐 key；要值就明说 .values()，要俩就 .items()。
05
list() 内置函数 四行手工循环 = 一行 list(...)
  • list(x)：把任何可遍历的东西逐个倒进一个新列表。判据：凡是能放进 for 的，都能喂给 list()。
  • groups.values() 返回的是视图对象不是列表，list() 一包才是正经 List[List[str]]。"建空列表→遍历→逐个 append"那四行，就是 list() 的内部实现——懂了原理之后，就该用一行的惯用写法。
  • ⚠️ 永远别拿内置名当变量名：for list in groups: 会把 list() 函数盖掉，之后再调就炸 'str' object is not callable。同理躲开 dict、str、sum、max。
list("abc")             # ['a','b','c']
list(range(4))          # [0,1,2,3]
list(groups)             # key 的列表(注意!)
list(groups.values())    # 值的列表 ← 本题答案
口诀惯用写法（idiom）的价值是让读者零思考。会走了，就要走得漂亮。
06
Review 复盘 这题被抓的四个毛病
  • 🔴 遍历字典拿了 key 当结果：返回了签名 ["aet","ant","abt"]，筐全扔了。→ .values()。
  • 🔴 拿 list 当循环变量，内置函数被覆盖——这次没炸只是运气。
  • 🟡 if/else 两个分支都在 append：建筐和塞词是两件事，建完筐统一塞，else 整个删掉。
  • 🟡 命名：full_List 混大小写，Python 用 snake_case；dist 装的是分组不是距离，应叫 groups。
心法修 bug 的顺序：先 🔴 致命再 🟡 风格——但 🟡 不是永远不管，是排在后面，收完才算真收工。
