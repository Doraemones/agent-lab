01
环境变量 .env 不是自动生效：先 load，再 getenv
  • load_dotenv() 是把 .env 里的键值加载进环境变量；find_dotenv() 是帮你从当前目录往上找 .env 文件。
  • OpenAI 初始化前必须先加载 .env，否则 os.getenv("DEEPSEEK_API_KEY") 读到的是 None。
  • api_key=load_dotenv 传进去的是函数本身；真正要传的是 os.getenv 读出来的字符串 key。
from dotenv import load_dotenv, find_dotenv
import os

load_dotenv(find_dotenv())

client = OpenAI(
    api_key=os.getenv("DEEPSEEK_API_KEY"),
    base_url="https://api.deepseek.com",
)
口诀load 负责加载，find 负责找文件，getenv 负责取值。顺序错了，key 就是空。
02
JSON Mode response_format 是开关，prompt 是合同
response_format
请求里加 {"type":"json_object"}，告诉 DeepSeek 这次要走 JSON 输出模式。它管的是输出形态，不等于业务字段一定完整。
prompt
提示词里必须出现 JSON / json，并给出字段说明和示例对象。不给 schema，模型更容易乱填字段名或吐空白。
temperature=0
抽取任务要稳定，设置 temperature=0。这是信息抽取，不是写作文，稳定比发散重要。
口诀response_format 开模式，prompt 定字段，示例锁格式。三件事缺一件，都可能不稳。
03
校验与重试 能 json.loads，才算这条 JD 合法
  • 模型返回的是字符串，不能肉眼觉得像 JSON 就算过。必须用 json.loads(raw) 真的解析一次。
  • 解析失败时最多重试 3 次；你这里更关键的一步是把坏输出作为 assistant 消息塞回去，再让模型修正。
  • 因为 temperature=0 很稳定，原样重发常会得到同一个坏结果；带上错误上下文，重试才有意义。
try:
    return json.loads(raw)
except json.JSONDecodeError as e:
    messages.append({"role": "assistant", "content": raw})
    messages.append({
        "role": "user",
        "content": "上面的输出不是合法 JSON，请只返回合法 JSON 对象。",
    })
口诀JSON mode 负责“尽量像 JSON”，json.loads 负责判死刑；失败重试要给模型看见自己错在哪。
04
数据落盘 内部英文 key，外部中文表头：FIELD_MAP 做边界
  • 脚本内部用 title / skills / salary / city / education，代码更稳定；写 CSV 时再映射成 岗位名、技能要求、薪资、城市、学历要求。
  • skills 是数组，CSV 单元格只能放文本，所以落盘前拼成 Python、MySQL、Redis 这种字符串。
  • 用 utf-8-sig 写文件，Excel 双击打开中文不乱码；文件不存在才写表头，追加时不重复。
FIELD_MAP = {
    "title": "岗位名",
    "skills": "技能要求",
    "salary": "薪资",
    "city": "城市",
    "education": "学历要求",
}
口诀模型输出给程序用，CSV 输出给人看。中间用 FIELD_MAP 做翻译层。
05
过关流程 真实 JD 是多行文本，按空行提交更顺手
1. 粘贴 JD多行复制，不用压成一行。
2. 空行提交单独敲空行，表示这一条结束。
3. LLM 抽取DeepSeek 返回 JSON 字符串。
4. loads 校验失败最多纠错重试 3 次。
5. 追加 CSV每成功一条立刻写入 jobs.csv。
  • 不用单个 input() 读 JD，因为招聘描述通常有岗位职责、任职要求、福利待遇等多段文本。
  • 每条成功后马上 save_to_csv([data])，中途程序挂了，也不会丢掉前面已经抽取好的结果。
  • Day 3 的真正验收不是“跑一次没报错”，而是从 BOSS / 牛客复制 20 条真实 JD，20/20 都能解析成合法 JSON 并落到 jobs.csv。
口诀输入按“条”收，输出按“条”验，落盘按“条”存。20 条真 JD 全绿才叫过关。
