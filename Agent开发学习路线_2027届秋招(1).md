# Agent 开发学习路线（2027届秋招 · 目标中厂）

> 你的起点：会 Python、调过 LLM API。今天是 2026年7月初。
> 秋招节点：提前批已开始，**正式批 8月启动**，10-11月面试密集期。
> 核心策略：**4周做出第一个项目 → 8月中旬前投出简历 → 边投边学边面**。不要等"学完"再投。

---

## 一、先回答你的疑问：两条路线的区别

| | Agent 应用开发工程师（工程向） | 算法岗（Agent训练/RL向） |
|---|---|---|
| 日常工作 | 用 LLM API + 框架搭建 agent 系统：工具调用、RAG、多智能体、上线部署 | 微调/后训练模型、RL 训练 agent、数据合成、评估 |
| 硬性门槛 | 项目 + 工程能力，无论文要求 | 通常要求论文、训练经验、扎实的 ML 功底 |
| 中厂岗位量 | **大**，几乎所有做 AI 产品的公司都招 | 少，多集中在有自研模型的公司 |
| 你现在准备 | **3个月完全来得及** | 来不及（除非你研究方向本就相关） |

**结论：走应用开发工程师路线。** 中厂这个岗位不卷学历和论文，卷的是"你能不能拿出一个真的能跑、有工程细节的 agent 项目"。

---

## 二、要学什么（技能地图）

### 第1层：立即要会（第1-2周）
- LLM API 进阶：**Function Calling / Tool Use**（agent 的基石）、结构化输出（JSON mode）、流式输出、多轮对话管理
- Prompt / 上下文工程：system prompt 设计、few-shot、上下文长度管理
- 工程基本功：Git、虚拟环境、`pytest` 基础

### 第2层：核心竞争力（第3-6周，面试必问）
- **RAG 全流程**：文档切分（chunking）、Embedding、向量库（Chroma/Milvus）、召回+重排（rerank）、评估（RAGAS）
- **Agent 经典范式**：ReAct、Plan-and-Solve、Reflection——要能手写简化版，不只是调框架
- **LangGraph**：目前招聘 JD 出现频率最高的框架；状态机、条件边、checkpoint、human-in-the-loop
- **MCP（Model Context Protocol）**：2025-2026 最热的 agent 工具协议，简历上的明显加分项
- 记忆管理：短期/长期记忆、对话压缩

### 第3层：工程化加分项（第7-8周）
- FastAPI 把 agent 包成服务（含 SSE 流式接口）
- Docker 部署、一个能访问的在线 demo
- 可观测与评估：Langfuse/LangSmith 追踪、构建评估集、bad case 分析
- 成本与延迟优化：缓存、模型分级调用（便宜模型做路由）

### 面试口头要懂（不用会训练）
Transformer/Attention 原理、SFT/LoRA 是什么、幻觉的成因与缓解、RAG vs 长上下文 vs 微调 的取舍。

## 三、12周执行计划（对齐秋招时间线）

| 周次 | 时间 | 学习重点 | 求职动作 |
|---|---|---|---|
| W1 | 7月上 | Function calling、结构化输出、prompt 工程；跑通 hello-agents 前3章 | 注册牛客/BOSS/各公司校招官网；**力扣启动，每天1题** |
| W2-W3 | 7月中下 | RAG 全流程，做出知识库问答 demo | 收集目标中厂名单（30-50家） |
| W4-W6 | 7月底-8月中 | ReAct 手写实现 → LangGraph → **项目一 v1 完成并部署** | **8月初简历 v1 完成，正式批开投** |
| W7-W8 | 8月下 | MCP 课程 + 给项目一加 MCP 工具、评估、观测；写好 README + 录 demo | 持续投递，开始笔试 |
| W9-W12 | 9月-10月初 | 转入面试模式：LLM八股 + 项目深挖问答 + 力扣中等题冲刺 | 面试密集期；可选做项目二 |

每天建议投入：学习 3-4 小时 + 力扣 30 分钟。毕业论文时间自己协调，但 8 月前项目一必须成型。

## 四、项目怎么做（简历的核心，决定你的面试）

### 项目一（必做）：一个垂直领域的多工具 Agent
选一个具体场景（如：论文调研助理、求职信息聚合 agent、某垂直行业问答），**不要做烂大街的纯 ChatPDF**。必须包含这些可被面试官深挖的要素：

- 多个真实工具调用（搜索、代码执行、数据库查询等），含**失败重试与降级逻辑**
- RAG 模块（有 rerank，说得出为什么这样切分）
- LangGraph 状态管理（画得出你的状态图）
- 流式输出 + FastAPI 服务化 + Docker 部署，**有可访问的 demo**
- 一个小评估集 + 指标（如任务成功率、幻觉率），README 里放数据

面试深挖点提前准备好答案：为什么这样设计状态机？工具调用失败怎么办？如何控制成本和延迟？哪个 bad case 让你改了方案？

### 项目二（加分，W9后有余力再做）
二选一：**写一个 MCP Server 并开源发布**（工作量小、辨识度高），或给 LangGraph/hello-agents 等开源项目提有效 PR。

## 六、避坑清单

1. **别从啃论文和学模型训练开始**——那是算法岗路线，你的时间不够。
2. **别只用 Coze/Dify 拖拽搭 agent**——中厂面试考代码，低代码平台只能作了解。
3. **别等"学完"再投简历**——8月正式批开始就投，面试本身是最好的学习反馈。
4. **别丢力扣**——中厂笔试仍有算法题，中等难度为主，每天1题坚持到10月够用。
5. **项目重深度不重数量**——一个能被追问三层还答得上来的项目，胜过三个 demo。
